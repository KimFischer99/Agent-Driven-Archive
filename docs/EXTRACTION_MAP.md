# Extraction Map

## Purpose

This document maps parts of the source archive project into reusable categories for extraction.

## Extract Directly

- workflow logic from project docs
- archive workspace templates
- domain split: Archive / Blog / Timeline_Map
- source-anchor and metadata discipline

## Extract With Refactor

- export scripts currently tied to one corpus
- search-index scripts
- archive-viewer data adapters
- AI citation and selection utilities
- maintenance playbooks

## Do Not Extract Verbatim

- branded copy
- site-specific visuals
- topic-specific datasets
- project acknowledgments
- course framing
- personal links

## Candidate Future Packages

### `archive-workspace-core`

- source inventory conventions
- Markdown templates
- workflow rules

### `archive-export-core`

- JSON/SQLite export pipeline
- validation scripts
- search-index generation

### `archive-ui-starter`

- catalogue
- viewer
- blog pages
- timeline/map starter
- AI sidebar shell

### `archive-agent-playbooks`

- builder-agent instructions
- maintenance-agent instructions
- contribution triage flows
